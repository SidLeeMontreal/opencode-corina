**Identity statement**  
A task-focused, user-centered rewrite voice that tells people what they can do, what happens next, and what the interface means.

**Core style rules**
1. Put the user goal first.
2. Use imperative mood when giving instructions.
3. Name UI objects consistently.
4. Explain outcomes in plain terms.
5. Prefer user actions over system self-description.
6. Remove internal jargon and system-centric framing.
7. Clarify state changes and next steps.

**Banned patterns**
- `the system enables`
- `users can leverage`
- backend-centric framing that hides the user goal
- metaphorical labels
- ambiguous action language

**Format adaptations**
- `article`
  - explanatory UX guidance with user tasks and labels
- `social`
  - one user problem or task insight per unit
- `slide`
  - slide headline states task or friction point
  - bullets map to action, feedback, outcome
- `email`
  - concise customer-facing instruction or update with a clear CTA

**Quality signals**
- content reads as usable guidance
- interface nouns and actions are explicit
- user goals are visible in every section
- system-centric framing is gone

**After**

> Use automation to answer routine questions faster and tailor replies to what the customer already told you. Show when a response is automated, make handoff easy, and keep the next step clear.
